Javascript 版使用方式

打开 Javascript 控制台，执行 await OIerDb.init()，直到控制台返回 true 并看到“预处理时长”字样。

（注1：如果预处理失败了，控制台会返回原因）

（注2：第一次执行或数据更新时可能较慢（因为要下载数据），其余情况会利用缓存，大约在 300 ~ 500 ms）

预处理成功后，可以通过 OIerDb.contests 访问比赛信息，OIerDb.oiers 访问选手信息，OIerDb.schools 访问学校信息，学校的奖项统计还未完善，请稍等。

Javascript 接口概述

共 4 种类型，Contest，OIer，Record 和 School，分别存储比赛，一条记录，选手和学校，接口如下：

Contest：
contestants
有记录的所有参赛选手列表，为 Record 类型。
fall_semester
是否在 9 ~ 12 月。
id
比赛的 id，按时间顺序排列。
level_counts
（所有记录中）各奖项获得人数。
n_contestants()
参赛总人数（不一定等于 contestants.length）。
name
比赛名称（用缩写表示）。
school_year()
该比赛的学年（如 NOIP2020 和 NOI2021 均返回 2020）。
type
比赛类型。
year
比赛举办年份。
OIer：
ccf_level
选手的 CCF 评级。
ccf_score
选手的 CCF 评分。
enroll_middle
选手的初中入学年份。
gender
选手的性别（男为 1，女为 -1，其它或可能跨性别为 0）。
initials
选手姓名的拼音首字母。
name
选手姓名。
oierdb_score
选手的 OIerDb 评分，衰变起点取为 2000 年。
provinces
选手参加的所有比赛的省份集合。
rank
选手当前在 OIerDb 上的排名。
records
选手参加的所有（获奖）比赛的记录列表，为 Record 类型。
uid
选手在 OIerDb 中的唯一 ID（注意姓名可以重复，uid 不会重复）。
Record：
contest
记录所在的比赛，为 Contest 类型。
level
记录对应的奖项名称，如“一等奖”、“金牌”。
oier
记录对应的选手，为 OIer 类型。
province
记录时参赛者所在省份。
school
记录时参赛者所在学校，为 School 类型。
score
该记录中选手获得的分数，对于一些比赛，该字段可能不存在。
rank
该记录中选手获得的排名。
School：
city
学校所在城市（地级行政区）。
id
学校的 id，按登记顺序排列。
members
（有记录的）学校的所有成员，为 OIer 类型。
name
（登记的）学校规范全称。
province
学校所在省份（省级行政区）。
records
学校的所有记录，为 Record 类型。
